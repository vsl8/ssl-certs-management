How Alert Rules Work

Alert rules define when notifications are sent based on certificate expiry dates.

  • Days Before Expiry: The system checks if a certificate expires within this many days
  • Channel: Where to send the notification (email, Slack, Teams, webhook)
  • Default Channels: If no specific channel is assigned, alerts go to all default channels
  • Smart Threshold Matching: Only the smallest matching threshold applies to each certificate
Example: If a certificate expires in 4 days, only the 7-day rule will trigger (not 15-day or 30-day rules).
Alerts are sent once per day per certificate to avoid spam.
